  S: CPU yükü neden önemli?

     Ancak şaşırtıcı şekilde HANDSHAKE daha hızlı, çünkü
     MLWE matematiksel olarak RSA'dan daha verimli hesaplanır.

  S: Veri transferi sırasında CPU farkı var mı?

     paket şifreleme CPU maliyeti TAMAMEN EŞİT.
     Ölçümlerdeki küçük farklar ağ/OS gürültüsünden kaynaklanır.

  S: Paket başına overhead neden tam 28 byte?
  C: 12 byte (AES-GCM IV/Nonce, replay koruması için)
     + 16 byte (GCM Authentication Tag, bütünlük doğrulama)
     = 28 byte sabit matematiksel zorunluluk.
